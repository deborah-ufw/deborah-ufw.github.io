If you examine the source code, you'll see files named "filename-gigwalk.xxx" and directories named "/brand/gigwalk/." In the specs, I'm proposing that these be paramaterized so that a {{brandname}} variable passed with the user's login correctly sets the directory and points to the brand's files. For example if the brand value is "crossmark", the app will point to "/brand/crossmark/" and "filename-crossmark.xxx" instead of Gigwalk. At the end of this process, both Crossmark and Gigwalk will be separately branded and optimized for a clean, efficient load of the correctly branded, responsive app.
